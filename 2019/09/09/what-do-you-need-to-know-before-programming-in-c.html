This tutorial is designed for the course cs139, C Programming and Algorithm Analysis of École d’Ingénieur SJTU-ParisTech. It’s not really an introduction about C programming language, but about some important things that we need to know before writing the real code.

You can follow this tutorial on Linux machines or macOS. But what about windows? Hmm…we don’t write code “sur les fenêtres”.

Write your first program

The premier thing and the principal work that you will do afterwards is writing the real code. First open the terminal on your machine, which is the program where you give your orders/commands to the computer like a chef. After opening the terminal, copy the line (except the symbol $) below to the terminal and press Enter:

$ echo 'This is my first Linux/Unix command!'

Voilà! You’ve made your premier step! Normally you will see something like this:

The line beginning with $ is the command that we execute in the terminal.

Enter the Documents directory, create a directory named cs139 and enter it:

$ cd Documents
$ mkdir cs139
$ cd cs139
  • cd: enter a directory
  • mkdir: make a directory

Make a C program source file named first_prog:

$ touch first_prog.c

Now open this file with any text editor that you like. (I prefer not to do any recommandation here to avoid starting a debate)

/* Preprocessor Section */
#include <stdio.h>                      // 'printf' needs this file

/* Program Body */ 
int main(void) {                        // This is where the program begins
    char french[] = "Bonjour";
    char german[] = "Guten Tag";
    char chinese[] = "ni hao";

    const unsigned int num_lang = 3;

    printf("We will use %d different languages to say hello:\n", num_lang);
    printf("%s(French), %s(German) and %s(Chinese)\n", french, german, chinese);
  
    return 0;                          // this statement terminates the program
}

Pay attention to the indentation even though the compiler doesn’t care about it and don’t leave out ; at the end of each line!

It is worth remembering that the code we are writing here is “text”, and machines can not understand it at all. Machines can only understand the sequences of “0” and “1”.

Compilation

A compiler is a program which translates source code from a high-level programming language (e.g., C, C++ or Swift) to a lower level language (e.g., assembly language or machine code).

For C programming language, two popular compilers exist:

  • GCC (GNU Compiler Collection)
  • Clang (C language family frontend for LLVM)

Get compilers on your own propre operating system

If you are working on the machines of IE lab, you can skip this part, since GCC has already been installed on them.

Ubuntu/Debian

For Ubuntu/Debian users, you can get GCC from your package manager, APT (Advanced Packaging Tool):

$ sudo apt update
$ sudo apt install build-essential

If you want to install Clang for your Debian/Ubuntu machine, please refer to this page: https://apt.llvm.org

macOS

For macOS users, Clang is recommended. Fun fact: Apple is one of the main contributors to the Clang project. Here, we will take a tour about how to install Clang without Xcode.

Install Command Line Tools:

$ xcode-select --install

Then follow the instructions on the screen.

Ubuntu/Debian/macOS

Even though you installed Clang on you OS, you can still use command gcc to compile your C programs. 👍

Check if the compiler is successfully installed:

$ gcc --version

On Debain it will output something like this:

Compile your first program

The simplest way to use gcc is:

gcc -o program file1.c file2.c ...
  • Flag -o specifies the name of the output file, which is the executable that the compiler creates.
  • Leaving out the -o will output the executable with the default name a.out.

Now let’s compile the program just written:

$ gcc -o first_prog first_prog.c

If you want to get the debugging information and all the warnings given by gcc, add the -Wall and -g flag:

gcc -g -Wall -o program file1.c file2.c ...

Running

Executing your program (binary file) is quite simple and easy:

$ ./first_prog

The program will output something as below if you do all the steps correctly:

Exercice 1

Try to add other languages in the program , re-compile and re-run your program.

Exercice 2

Try to get the length of the string french: add those statements after the char french[] = "Bonjour":

const unsigned int len = sizeof(french);
printf("Bonjour is of length %d\n", len);

What result do you get? Is it strange? But why?

Exercice 3

Add these two lines before the return statement:

char out_of_index = french[9];
printf("The value at index 9 is: %x\n", out_of_index);

Re-compile it with the warning option enabled, what information or warning do you get from the compiler? Is it same between Clang and GCC? Re-run it, does the program fail?

The C Compiler doesn’t prevent you from making mistakes like “out of index” and many others ☹️!!!

Exercice* 4

Create a new file:

#include<stdio.h>
#include<stdlib.h>

int main(int argc, char* argv[]) {
    int i;
    int total = 0;
    for (i = 0; i <= argc; i++) {
        total += atoi(argv[i]);
    }
    printf("The total is %d\n", total);
    return 0;
}

Compile and run it with the command below:

$ gcc -g -o sum sum.c
$ ./sum 3 4 5

Which runtime error have you encountered?

If you don’t understand at all what it is or how to solve it, DO NOT WORRY! We will come back to it soon, maybe with Prof Lu? This example just wants to show you that this kind of error will happen very often later!
